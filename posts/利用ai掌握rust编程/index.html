利用AI掌握Rust编程

AI 时代的技术进阶：快速深入 Rust 编程之道

Dec 21, 2024
阅读时长: 3 分钟

缘起

看了一眼，这篇博客的创建时间，2024年12月21日，八天前，已经想不起因为什么原因想要再学习一门语言了，似乎可能因为接触区块链Web3与加密货币比较多的信息，又想着能不能入门Web3，把握这个未来可能的风口，于是想要学习一门新的语言，问了ChatGPT，问它未来最有潜力的编程语言是什么，它输出的第一个语言是Rust，可能因为种种这些原因吧。

考虑到自己已经不算是新手，有其他很多编程语言的基础，那么再想如果再学习其他语言，有更好的方式么？

尤其是在这个AI迅猛发展的时代，这样一个时代，似乎知识变得触手可得，相比于早十年互联网时代，AI时代不仅仅获取信息更加方便，而且AI相比于传统的互联网已经可以快速整合知识，甚至于快速生成一篇论文或者报告。

这个时代，或许对于每个人来说，都难免产生焦虑与恐慌，毕竟每个人努力三四年学习的知识与技能，在当下AI可能半年时间就超过一个正常人三四年的练习的水平了，这或许是焦虑的来源；此外，恐慌来源于AI快速发展，似乎越来越像人，拥有人类所谓的“自由意识”或“灵魂”，反观人类本身，似乎活得越来越像机器人或游戏中的NPC。

总结，机器人AI越来越像人，人越来越像机器人，人与机器人之间的界限越来越模糊。

2022年年底，ChatGPT发布，这年年底在网上冲浪，有幸算是较早接触到这样的自然语言领域的大模型，这两三年虽然说在人工智能领域读研究生，也看过非常多的AI方向论文，略微了解各种模型背后的原理，然而当ChatGPT发布时，还是给了我非常大的震撼，似乎小时候一直比较感兴趣的机器人，真的诞生了，它真的能够流畅自然的像人一样进行对话，以至于有那么多次觉得它真的有了人的意识，但是理智告诉我它还是一个基于概率论下的产物，它再尽量猜测下一个词语，它给我反馈的不过是它每次计算得到概率最大的结果。

但是，这样的科技产物还是让我感到未来已经到来，似乎电影中小说里描述的未来机器人已经诞生，当时暗暗心想这个AI为了会影响整个世界，重塑我们的观念。

话题说远了，回到前面的问题，如果再想学习一门语言，有更好的方式么？

其实在互联网时代，每个人获取知识都已经非常便捷了，真的是量大管饱，各种电子书，各种教程，各种视频教程，可以说这个时代只有你想学，肯花时间，肯下功夫，就能学到非常多知识和技能，然而，借助AI再学习一个新的知识，一门新的编程语言，又能起到什么作用呢？

思来想去，如果用AI获取知识，那这样的方式和互联网时代搜索知识，并没有本质的区别，只是更加方便和快速了，反思之前的教育经过和学习经历，学习一个新知识，最大的痛苦应该是自己的疑问和困惑没有得到解决，试想我们每个人有不同的知识背景，不同的熟悉领域，在学习一个新知识时，每个人都难免有着或大或小的对于新知识的问题或困惑，以往的经验是我们可以咨询一个老师，咨询同学，或者问问自己家长，但是应该每个人都有过自己耻于问别人问题的时刻吧，虽然古云“不耻下问”，但是真正能够对每个问题都不耻下问的人，也算是少数人吧。

如今，AI不仅能够获取信息，它最重要一点就是可以针对你的问题进行回答，而且试图检索它见过的所有数据库与知识库，进行回答。那么，我突然明白AI赋予人的一个最大的用途，就是答疑解惑，并且像一个玄幻小说的智者一样，随时在你的身上，回答自己的困惑与并给出合理的建议。

明确了AI的作用之外，还有另外一个问题，那就是人学习到的知识会随着时间慢慢忘却，这样的问题怎么解决呢？三年前，一直在思考这样的问题，现代社会信息量非常庞大，如何管理自己的知识呢？靠人脑记忆所有的知识已经完全不可能了。写到这里，先问问AI怎么回答的，ChatGPT回答得非常全面，“使用间隔重复法，主动回忆法，多感官参与学习，关联与应用，教别人或者输出知识，管理学习状态，重视复盘与整合，持续学习与循环暴露”。这些方法说实话自己看过相关的书籍都或多或少的熟知，然而说实话，自己执行力并没有那么强，虽说知道了很多，然而并没有做到“知行合一”。

中午，百无聊赖时，翻起和渊老师的《成为学习高手：清华博士的高效学习秘籍》，里面又提到“输出”方法，里面说的案例，其实也是自己熟知的，学生回到家后将白天在课堂学习到的知识讲给自己的家长，不知不觉中，学生的成绩得到了提升。另外，昨天看到一个博主的博客文章，里面说到“只要会思考就能写作”，突然想到自己也会思考，为什么想了两三年的问题“如何学习写作”，自己没能做到呢？一来自己把写作这个事情看得太过于神圣，太过于严肃，似乎要成为作家，似乎要是文科生，才能写作自如；二来自己写点东西总觉得很矫情，没有什么价值。三来觉得自己的文笔不够好，表达内容总是思来想去，期待有更好的表达方式。

昨天在看了几篇博客文章，其内容也比较偏口语化，像是与人聊天一样的文字表达，但是读起来却让人感到舒服与愉快，仔细想想虽然文字不像是ChatGPT等AI输出的内容表达得正式与书面化，但是这样的文字无形之间又让读者感到亲近感，尤其这些年阅读非常多的大牛作家的作品，发现他们的文字也都非常容易读，平易近人。这样上面的三个阻碍自己写作的心里障碍似乎都想通了。

这两天折腾给博客和编辑器VS Code都换上新的字体霞鹜文楷LXGW WenKai，确实很好看，以上种种原因，再加上自己或许需要一个方式来表达，于是产生了这个缘分，写下来这些内容。

use std::collections::HashMap;

fn main() {
    // 使用 Vec 存储一组数字，展示所有权与可变性
    let mut numbers = vec![1, 2, 3, 4, 5];
    
    // 使用闭包对 Vec 中的每个元素进行平方运算
    numbers.iter_mut().for_each(|x| *x *= *x);
    println!("Squared numbers: {:?}", numbers);

    // 使用模式匹配从 Vec 中查找特定值
    match numbers.iter().find(|&&x| x == 16) {
        Some(&value) => println!("Found the value: {}", value),
        None => println!("Value not found"),
    }

    // 使用 HashMap 存储学生姓名和分数，展示集合类型
    let mut scores = HashMap::new();
    scores.insert("Alice", 90);
    scores.insert("Bob", 85);
    scores.insert("Charlie", 92);

    // 使用模式匹配和 Option 处理可能不存在的值
    if let Some(&score) = scores.get("Alice") {
        println!("Alice's score: {}", score);
    } else {
        println!("Alice's score not found");
    }

    // 演示错误处理，尝试解析字符串为数字
    let input = "42";
    match input.parse::<i32>() {
        Ok(number) => println!("Parsed number: {}", number),
        Err(e) => println!("Failed to parse number: {}", e),
    }

    // 使用枚举定义和模式匹配处理不同的状态
    enum TrafficLight {
        Red,
        Yellow,
        Green,
    }

    let light = TrafficLight::Green;
    match light {
        TrafficLight::Red => println!("Stop!"),
        TrafficLight::Yellow => println!("Caution!"),
        TrafficLight::Green => println!("Go!"),
    }
}

Rust语言基础知识点

  1. 在 Rust 中，函数的返回值等同于函数体最后一个表达式的值。使用 return 关键字和指定值，可以从函数中提前返回；但大部分函数隐式返回最后一个表达式。

  2. 在Rust中，引用（&T）类似于 C/C++ 中的指针或引用，但更安全。引用允许你访问变量的值，而不需要拥有它的所有权。特点是：1）不可空：Rust 没有 nullptr，引用总是指向一个有效的值。2）自动解引用：无需手动 * 操作，引用可以直接使用。3）默认不可变：&T 是不可变引用，值不能被修改。4）可变引用：&mut T 是可变引用，允许修改值。

fn main() {
    let array = [1, 2, 3, 4, 5];

    // 使用 `to_vec` 方法
    let vec1 = array.to_vec();
    println!("{:?}", vec1); // 输出: [1, 2, 3, 4, 5]

    // 使用 `Vec::from`
    let vec2 = Vec::from(array);

}
