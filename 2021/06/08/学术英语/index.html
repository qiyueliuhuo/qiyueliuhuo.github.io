section 1

The theme of my speech is Code volume optimization technology for embedded applications. Next, I will introduce from the following several aspects.

Whether it's an electronic product that is used in everyday life, or an office device that is used in all walks of life, we can see embedded systems everywhere. Embedded systems also play a huge role in national defense. With the development of computer and semiconductor technology, embedded software are becoming more and more complex and huge, software optimization still faces two problems, first, the limited storage, the embedded systems with limited resources, plus more and more complex software, requiring more storage space; Second, code complexity, with a high code execution overhead, people pursue higher performance.

In order to solve the problem of limited storage, the compiler is often using the optimized technology which is based on code volume, The compiler performs optimization techniques with the goal of executable code volume. At this point, the compiler prohibits some optimizations which may increase the code volume. The subject takes the compiler which is widely used in the embedded domain as the object of the study, using the program execution section information,optimizing executable code size.

section 3

The main work and achievements of the paper include: 1．using of Intel Vtune tool to analyze the performance difference, as the basis of the optimization. 2．Proposing methods for optimizing the volume of code based on hot-spot code. The program performs a section called "hot" and "cold" code. The "hot" code accounts for more than 90 percent of the execution time of the program, but the code size is often just a small part of the code; Upon this property, the "hot"and"cold" code use different optimizations. 3．using the different compiler options upon "hot"and"cold" code.
